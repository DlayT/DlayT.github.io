关于js 脚本缓存的问题

Posted on 2018-12-21

关于禁止js文件缓存问题

对于js 文件被缓存的问题,会导致修改文件上传后,用户的浏览器使用的还是本地缓存的js 代码,导致用户体验到的不一致, 需要强制刷新之后, 重新取服务器获取最新的js代码, 但是不能让用户每次都取这样操作.

一般的解决方案:

  • 加版本号 
添加版本号的方案:这个方案是属于业界比较成熟的方案 但是有个缺点,这个版本号怎么做版本, 不可能修改一次就更新一次版本号, 故而版本号管理是个问题
  • 加发布时间 
这个方案是可行的  不需要系统的时间,而且不需要每个js文件都添加参数  每次修改后 去手动修改参数  但钱的时间
  • 开发过程中添加时间戳或者添加随机数
    <script type=”text/javascript” src=”/js/demo.js?t=’date(“Y-m-d”)’” > 使用日期，重新请求服务器。
    使用时间戳
添加时间戳的方案  可以解决缓存问题, 不过缺点也是很明显, 因为引入时间戳后会导致浏览器每次都会去服务端请求js文件资源,  这样会很消耗资源

添加的方法:

  • 直接写入页面
    document.write( " <script src='demo.js?rnd= " + Math.random() + " '></s " + " cript> " )

  • 动态追加脚本标签 然后填入地址

    var js = document.createElement( " script " )+
    js.src = " test.js " + Math.random()+
    document.body.appendChild(js)

  • 手动修改后面的参数
    <script src='demo.js?t=20180428'></script>

后来又找到一个方法,那就是用自动化构建工具  比如gulp, 每次修改提交后都构建一次, 修改了文件名,  这样每次构建提交之后 都会实时的更新被缓存的文件

`   
    runSequence = require('run-sequence'),  
    rev = require('gulp-rev'),   // 清楚缓存
    revCollector = require('gulp-rev-collector');   // 替换指定文件
`


总结: 目前能想到的就是这样写方法  ,如果有更好的方法 再回来补充
